## 1. 代码审查 — 发现的问题与修复

### 1.1 严重BUG (Critical)

#### BUG-1: TCP坐标单位错误 (m写入为mm)

| 项目 | 内容 |
|------|------|
| **文件** | `test/testS50PalletizingSO.cpp` |
| **位置** | `executeP2P()` 和 `executeRRTStar()` 函数 |
| **问题** | FK2返回**米**, 但CSV header标注`tcpX_mm,tcpY_mm,tcpZ_mm`, 轨迹文件列格式规范也要求mm。原代码直接写入`rp.tcpPose.X`(米), 导致输出值差1000倍 |
| **影响** | 所有输出数据的TCP坐标错误, MATLAB/Python可视化位置偏差1000倍 |
| **修复** | 所有TCP坐标输出添加`×1000`转换: `rp.tcpPose.X*1000`, `tc.X*1000` 等 |
| **严重性** | **Critical** — 影响所有下游分析 |

### 1.2 违反"无魔法数字"规则 (Magic Numbers)

#### MN-1: EllipsoidSamplerSO 使用硬编码默认参数

| 项目 | 内容 |
|------|------|
| **文件** | `include/PalletizingPlanner/PathPlannerSO.hpp` (原第278行) |
| **问题** | 构造函数中创建`RobotDHParams defaultParams`获取关节限位, 而非使用机器人实际参数 |
| **违反规则** | copilot-instructions.md: "所有常量必须来自参数文件或RobotDHParams结构体, 不允许硬编码" |
| **修复** | 构造函数改为接受`const RobotDHParams& params`参数, 调用处传入`robot_.getParams()` |

#### MN-2: CollisionReportSO.isSafe() 硬编码裕度

| 项目 | 内容 |
|------|------|
| **文件** | `include/PalletizingPlanner/CollisionCheckerSO.hpp` (原第121行) |
| **问题** | `isSafe(double marginMm = 10.0)` 使用字面量`10.0`, 而非引用`S50CollisionGeometry`常量 |
| **修复** | 改为 `isSafe(double marginMm = S50CollisionGeometry::defaultSafetyMarginMm)` |

#### MN-3: S曲线峰值速度因子 1.875 未命名

| 项目 | 内容 |
|------|------|
| **文件** | `include/PalletizingPlanner/TimeParameterization.hpp` (原第329行) |
| **问题** | `segmentTime *= 1.875;` 直接使用数字, 虽有注释但不够规范 |
| **修复** | 提取为命名常量 `constexpr double kSmootherStepPeakVelocityFactor = 30.0 / 16.0;` |

#### MN-4: ToolConfig 参数缺少来源文档

| 项目 | 内容 |
|------|------|
| **文件** | `include/PalletizingPlanner/RobotModel.hpp` (原第171-197行) |
| **问题** | `toolRadius=0.05`, `approachOffset=0.1`, `retractOffset=0.15` 等值无来源说明 |
| **修复** | 添加参数来源注释: "吸盘安装偏移: 法兰→吸盘中心150mm", "吸盘外径~160mm, 包络半径取80mm" 等 |

### 1.3 单位不一致 (Unit Inconsistency)

#### UI-1: PlannerConfig.safetyMargin 单位模糊

| 项目 | 内容 |
|------|------|
| **文件** | `include/PalletizingPlanner/Types.hpp` (第376行) |
| **问题** | `safetyMargin = 0.01` 注释标注`[m]`, 但`CollisionCheckerSO`全部使用mm单位 |
| **修复** | 补充注释: `(注: CollisionCheckerSO使用mm单位, 10mm=0.01m)` |

### 1.4 碰撞模型不完整

#### CM-1: 框架缺少顶梁和侧挡板

| 项目 | 内容 |
|------|------|
| **文件** | `test/testS50PalletizingSO.cpp` |
| **问题** | 指导文件要求: "4根近端立柱 + 2根顶梁 + 2块侧挡板", 但原代码仅添加4根立柱(envId 30-33) |
| **修复** | 新增2根顶梁(envId 20-21, 连接近端/远端立柱顶部), 2块侧挡板(envId 22-23, 框架两侧中间高度) |

### 1.5 测试覆盖不足

#### TC-1: 无TCP水平约束测试

| 项目 | 内容 |
|------|------|
| **文件** | `test/testTrajectoryOptimality.cpp` |
| **问题** | 6套测试(A-F)全部设置`constrainTcpHorizontal = false`, 无法验证码垛核心功能 |
| **修复** | 新增测试G: 4组TCP水平约束案例, 验证路径上每个点的TCP Z轴偏角 ≤ 30° |

### 1.6 文档/注释错误

#### DC-1: 误导性注释与代码矛盾

| 项目 | 内容 |

| **文件** | `test/testS50PalletizingSO.cpp` (原第237行) |
