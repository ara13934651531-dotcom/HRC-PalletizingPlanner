# HR_S50-2000 码垛场景完整描述

> **用途**: 机械臂避障路径优化 · 仿真环境搭建 · 碰撞检测配置  
> **输入**: TCP 起始位姿 (X, Y, Z, A, B, C)  
> **输出**: 无碰撞的关节空间轨迹, 终止于 TCP 目标位姿  
> **版本**: v3.0 &emsp; **日期**: 2026-03  
> **机器人**: HR_S50-2000 (广东华数机器人有限公司 / HansRobot)

---

## 概要 (Executive Summary)

本文档完整描述了 HR_S50-2000 6-DOF 工业机器人在码垛工作站中的场景配置, 包含:

- **机器人**: 6-DOF, 最大负载 50kg, 臂展 ~2000mm, 安装于 800mm 高白色电控柜上
- **码垛框架**: 1200×650×2000mm 蓝色方管钢结构笼体, 三面金属丝网格封闭, 前方开口
- **传送带**: 550×3500mm 滚筒式, 位于机器人右侧 (X=750mm), 皮带面约与基座齐平
- **末端执行器**: 墨绿色真空吸盘阵列 (2×2 大吸盘), 法兰安装, 带蓝色气管
- **任务**: 从传送带吸取 350×280×250mm 纸箱, 放入框架内托盘上, 4位×3层=12箱
- **碰撞系统**: 5个机器人自碰撞体 + 13个静态环境障碍 (建议增至 15 个) + 动态工具球/已放箱球
- **关键挑战**: 框架三面网格封闭仅前方开口, 顶梁限高, 工具碰撞球过框架入口间隙紧凑, J3 构型切换

文档涵盖完整坐标、碰撞参数、API 接口、风险分析和路径规划建议, 可直接用于代码生成和仿真环境搭建。

> ⚠️ **本文档基于 3 张实物照片 (2026.02.25) + testS50_Palletizing_v15.m 仿真代码 + C API 头文件联合标定。**
>
> **数据来源优先级**: ① 代码常量 (精确值) → ② URDF/JSON 配置文件 → ③ 照片目视估算 (±50mm 精度)。
> **设计原则**: 模型准确但简洁高效, 以最少碰撞体覆盖关键风险, 兼顾泛化性和求解速度。

## 1. 坐标系约定

### 1.1 世界坐标系 (World Frame)

| 属性 | 定义 |
|------|------|
| **原点** | 地面上, 机器人基座正下方投影处 |
| **X 轴** | 水平, 指向传送带方向 (右) |
| **Y 轴** | 水平, 指向码垛框架方向 (前) |
| **Z 轴** | 垂直向上 |
| **机器人基座 Z** | Z_world = 800 mm (电箱顶面) |

### 1.2 基座坐标系 (Base Frame) — **主坐标系**

| 属性 | 定义 |
|------|------|
| **原点** | 机器人基座法兰中心, 即世界坐标系原点 + (0, 0, 800) mm |
| **轴向** | 与世界坐标系平行, 仅 Z 偏移 |
| **地面 Z** | Z_base = −800 mm |
| **所有碰撞坐标** | 以基座坐标系为参考, 单位 mm |
| **FK/IK** | 输入: 关节角 (deg), 输出: TCP位姿 (mm + deg) |

> ⚠️ **所有后续坐标均在基座坐标系中, 除非特别标注。**

### 1.3 坐标系转换

```
P_base = P_world − [0, 0, 800]
P_world = P_base + [0, 0, 800]
```

### 2.2 DH 参数

#### C API 接口参数 (mm)

| 参数 | d1 | d2 | d3 | d4 | d5 | d6 | a2 | a3 |
|------|------|------|------|------|------|------|------|------|
| **值** | 296.5 | 336.2 | 239.0 | 158.5 | 158.5 | 134.5 | 900.0 | 941.5 |

```c

```

#### URDF / 渲染参数 (m)

| 参数 | d1 | d2 | d3 | d4 | d5 | d6 | a2 | a3 |
|------|------|------|------|------|------|------|------|------|
| **值** | 0.2833 | 0.3345 | 0.239 | 0.1585 | 0.1585 | 0.1345 | 0.900 | 0.9415 |

#### 结构含义

- **d1** = 基座高度 (基座法兰到 J1 旋转轴)
- **d2** = J1 轴到 J2 轴的偏移 (肩部)
- **d3** = J2 轴到肘部偏移
- **a2** = 大臂长度 (J2→J3)
- **a3** = 小臂长度 (J3→J4)
- **d4, d5** = 腕部偏移
- **d6** = J6 到 TCP 的距离

## 3. 场景总览与空间布局

### 3.1 实物场景描述 (基于实际照片)

实际场景为工业车间内的码垛工作站 (照片拍摄于 2026.02.25, 三个角度):

**机器人:**
- **白色 HR_S50-2000** (HUAYAN 品牌铭牌可见) 安装在 **白色钣金电控柜** 顶部
- 电控柜前面板有: **绿色 LCD 显示屏** (状态监控)、**红色急停蘑菇头按钮** (左下方)
- 机器人本体: 白色连杆 + 黑色关节环 (电机外壳), 黑色线缆/气管沿臂外侧走线
- 总高约 1.6 m (地面到基座法兰), 与电控柜高度 (0.8m) 之和

**蓝色码垛框架 (Safety Cage):**
- **方管钢结构**, 强电蓝色 (RAL 5012 近似) 喷漆焊接框架
- **左、右、后三面** 装有 **菱形/方孔金属丝网格防护板** (网孔 ~50×50mm, 线径 ~3-4mm)
- **前面 (靠机器人一侧)** 为主要开口, 有部分门框结构和 **橙色安全卡扣/锁紧带** (照片可见)
- **顶部** 四根横梁形成矩形封顶 (X向前后各1根 + Y向左右各1根)
- **无中部水平横梁** (实物确认: 框架为简洁的四柱+顶梁+底梁结构)
- **底部水平横梁** (接近地面, ~Z_world≈50mm): 沿 X 方向连接前后立柱底端
- 框架内部放置蓝色托盘/平台, 其上码放纸箱

**传送带:**
- 位于机器人右侧方, 大致沿 Y 方向延伸
- **滚筒式传送带**: 可见多根平行滚筒 + 上方皮带
- 上有纸箱产品 (照片可见 Neurio 品牌纸箱, 棕色/彩色印刷)
- 白色钢管腿架支撑, 皮带面高度约与机器人基座齐平

**末端执行器 (真空吸盘):**
- **墨绿色/深青色** 吸盘模块 (照片3 清晰可见, 非蓝色)
- 铝合金安装板 + **2×2 大圆形吸盘阵列** (覆盖箱子顶面主要区域)
- 蓝色气管从法兰沿臂侧走线连接
- 有品牌铭牌 (疑似 新芝量子/森泰科 等真空元件品牌)
- 吸盘朝下安装, 从箱子顶面吸取

**周边环境:**
- 工作站前方设有 **红色安全警示绳/栏杆** (不锈钢立柱+红绳, 照片1可见)
- 机器人旁有 **触摸屏操作面板** (平板式, 照片2可见, 用于HMI操控)
- 车间地面为灰色工业地毯/防静电地板
- 后方 (机器人背侧) 有杂物堆积 (红色新年礼盒等, 非工作相关, 不影响码垛路径)

**尺寸估算方法论:**
所有几何参数的来源分三级:
1. **代码精确值** (±0mm): DH参数、碰撞体坐标、框架/电箱/传送带尺寸均直接读取 `testS50_Palletizing_v15.m` 代码常量 (§1 参数定义段, lines 44-115)
2. **配置文件** (±5mm): 碰撞体几何来自 `S50_collision.json`, URDF 变换来自 `.urdf.xacro`
3. **照片估算** (±50mm): 吸盘尺寸/颜色基于照片3近距离拍摄

#### 碰撞通道

| 通道 | Body Pair | 默认状态 |
|------|-----------|---------|
| 肘通道 | (Elbow, 其他) | 开启 |
| 腕通道 | (Wrist, 其他) | 开启 |
| 工具通道 | (Tool, 其他) | 开启 |

> 建议: **始终开启全部 3 个碰撞通道**, 以确保完整的自碰撞检测。

### 5.2 自碰撞检测原理

碰撞检测库 (libHRCInterface) 在每个控制周期:
1. 根据当前关节角更新所有碰撞体的世界坐标 (通过 FK)
2. 计算所有启用碰撞通道的 body pair 之间的最小距离
3. 如最小距离 ≤ 0, 返回碰撞标志 + 碰撞对 ID + 穿透距离

返回值:
- `colliderPair[2]`: 碰撞的两个 body ID
- `distance`: 最近距离 (正=无碰撞, 负=穿透)

### 5.3 连杆-环境碰撞

每个连杆 (含工具) 与全部已注册环境障碍物进行距离计算。连杆-环境碰撞开关可独立控制 (7 个: Base, Lower, Elbow, Upper, Wrist, Tool1, Tool2)。

### 5.4 碰撞建模简化原则

| 原则 | 说明 |
|------|------|
| **最少碰撞体** | 以 15 个碰撞体覆盖全场景关键风险, 减少每次碰撞检测的计算量 |
| **几何包络** | 优先使用 Capsule/Ball 简单体, 避免 Mesh 碰撞 (计算慢 10-100×) |
| **忽略低风险** | 底梁 (Z=−750, 远低于工作区)、地面杂物等不建模 |
| **等效约束** | 网格防护板等效为 3 个半平面不等式, 不建为碰撞体 |
| **过包络容忍** | 工具碰撞球 r=225mm 包裹吸盘+箱子, 略有过包络, 换取模型简单 |
| **泛化优先** | 避免过度拟合特定几何, 框架/电箱/传送带参数化定义, 可快速适配类似场景 |

## 9. 关键约束与危险区域

### 9.1 碰撞风险等级图

```
最高风险 ★★★:
  ├── Seg 4 (PkApp→PlApp): 携带箱子穿越电箱→框架过渡区
  │   → 工具碰撞球(r=225) vs 框架近端柱(col1/col2) + 电箱后面板(envId 13)
  │   → 入口通道仅 1100mm 宽 (两近端柱间), 工具球直径 450mm, 净余 650mm
  ├── Seg 5 (PlApp→Place) @ L3: 框架内第三层下降
  │   → 距顶梁仅 ~360mm 净间隙, 过渡路径需远离顶梁
  └── Seg 7 (PlApp→HOME) @ 后排: 从框架远端撤出
      → J1 大角度旋转, 可能扫过立柱

高风险 ★★:
  ├── 后排放置 (pos2/pos3): Y≈780, 臂展接近极限
  │   → J2/J3 接近限位, 可用姿态空间小
  ├── L2 层 J3>0 配置 (box 2,5,7,10): 手肘翻转
  │   → 自碰撞风险 (body pair [4,2]: UpperArm vs LowerArm)
  └── 已放置箱子累积: 后期 (box 9-11) 框架内已有 9+ 个碰撞球
      → 可用空间急剧减小

中等风险 ★:
  ├── 拾取段: 传送带上方操作, 空间相对开阔
  ├── L1 层前排 vs 立柱: 净间隙≥161mm, 安全但非宽裕
  └── L3 层放置 vs 顶梁: 净间隙 360mm+, 放置位安全, 过渡路径需注意
```

### 9.2 量化间隙分析

> **计算方法**: 以下距离为 **工具碰撞球中心** (TCP + [0,0,−125]) 到 **最近障碍物碰撞体轴线** 的 3D 最短距离。净间隙 = 距离 − r_obstacle − r_tool (其中 r_tool=225, r_col/r_bar=50)。

| FrL-L1 | (−179, 506, 337) | col1 | 441 | **166** | ✅ 安全 |
| FrR-L1 | (152, 515, 337) | col2 | 469 | **194** | ✅ 安全 |

### 9.3 J3 正负切换问题

码垛过程中 J3 会在两个配置之间切换:

| 配置 | J3 值 | 使用场景 | 特点 |
|------|-------|---------|------|
| **J3 < 0** | −135° ~ −163° | L1/L3层大部分, 前排 | 常规构型, 肘朝前上方 |
| **J3 > 0** | +125° ~ +130° | L2层部分 (box 2,5,7,10), L3层部分 | 翻转构型, 肘朝后下方 |

> ⚠️ **切换 J3 符号时**: 机器人需经过构型切换 (可能需要大幅度关节运动), 路径规划器必须确保过渡路径无碰撞。曾发现 J3 = −165° 会导致 body pair [4,2] (UpperArm vs LowerArm) 自碰撞, 已限制为 −163° 最小值。

### 9.5 环境碰撞 "热区"

以下区域是环境碰撞的高发地带, 路径优化时需特别加权:

1. **框架入口** (Y = 325~425, X = −600~+600):
   - 近端两根立柱之间, 机器人臂从此进入码垛区
   - 电箱后面板 (envId 13, Y=325, r=80mm) 紧邻

2. **框架顶部** (Z = 1100~1300):
   - 两根顶梁 (Z=1200, r=50mm) 限制了上方空间
   - L3 层操作时工具碰撞球可能触顶

3. **电箱-传送带过渡区** (X = 275~475, Y = −325~+325):
   - 电箱右面板 (envId 11, r=80mm) 与传送带左轨 (envId 15, r=80mm) 之间
   - 物理间距 200mm, 扣双侧碰撞半径后净间隙仅 40mm, 任何连杆均无法通过

| `joint_limits` | double[6][2] | deg | 关节限位 [下限, 上限] |

### 11.3 优化目标

1. **安全性** (硬约束): 全程无自碰撞、无环境碰撞
2. **关节限位** (硬约束): 所有路径点满足关节限位
3. **路径长度** (软约束): 最小化关节空间路径长度 Σ‖Δq‖
4. **安全裕度** (软约束): 最大化与障碍物的最小距离
5. **平滑性** (软约束): 最小化加速度跳变

### 11.4 建议的路径优化管线

```
TCP_start, TCP_goal
        │
        ▼
   ┌─────────┐
   │  IK 求解  │ → q_start, q_goal (可能有多解)
   └────┬────┘
        │
        ▼
   ┌──────────────┐
   │ 采样式路径规划 │  RRT* / PRM / BiRRT
   │  (关节空间)   │  碰撞检测: hrc_checkSelfCollision
   └────┬─────────┘
        │
        ▼
   ┌──────────────┐
   │  路径优化/平滑 │  CHOMP / TrajOpt / STOMP
   └────┬─────────┘
        │
        ▼
   ┌──────────────┐
   │  轨迹时间参数化 │  TOPP-RA
   └────┬─────────┘
        │
        ▼
   collision-free trajectory
```

### 11.5 IK 多解与构型选择

6-DOF 机器人的 IK 通常有多组解 (最多 8 解)。本场景中码垛位使用了两种主要构型:

| 构型 | J1 范围 | J3 符号 | 使用场景 | 特点 |
|------|---------|--------|---------|------|
| **A (常规)** | +96° ~ +138° | J3 < 0 | L1, L3 大部分 | 肘朝上, 路径较短 |
| **B (翻转)** | −138° ~ −96° | J3 > 0 | L2 层部分 | 肘朝下, 需 J1 反转 ≈180° |

> **IK 选择建议**: 路径规划器收到 IK 多解后, 应选择与当前构型 **关节空间距离最小** 的解, 避免构型跳变。若必须切换构型 (如从 A→B), 应插入中间路径点确保过渡安全。

> **J4=0, J6=0 规律**: 当前所有 12 个放置位均使用 J4=0, J5=90°, J6=0°, 这意味着末端始终保持固定姿态 (竖直朝下), 简化了腕部自碰撞分析。若未来需要非竖直姿态放置, 需重新评估腕部碰撞风险。

### 11.6 推荐的路径段策略 (Via-Points)

针对码垛场景, 建议对 Seg 4 (拾取接近 → 放置接近) 使用中间路径点:

```
拾取接近位 (PkApp)     携带箱子, 传送带上方
       │
       ▼
安全提升点 (LiftPt)     Z ≈ 600~800mm, 远离电箱/传送带
       │
       ▼
框架入口点 (EntryPt)    Y ≈ 300, Z ≈ 600~1000, 在两近端柱之间
       │
       ▼
放置接近位 (PlApp)      框架内部, 略高于放置位
```

**推荐安全区 (Safe Zone)** 用于中间路径点:
- X ∈ [−400, +400] mm (框架内侧两柱之间中央区域)
- Y ∈ [0, 350] mm (电箱上方到框架入口前)
- Z ∈ [600, 1100] mm (足够高, 远离电箱和传送带, 低于顶梁)

**框架入口走廊约束** (从前方开口进入框架时):
- **水平通道**: X ∈ [−550, +550] mm (两近端柱间, 扣碰撞半径 50mm)
- **下限 Z**: 无硬限制 (框架无中部横梁, 托盘顶面 Z=200 为操作面)
- **上限 Z**: ≤ 1150mm (低于前X顶梁 Z=1200 − 碰撞半径 50mm)
- **可用窗口**: 1100mm × 950mm (X×Z), 从 Z=200 到 Z=1150, 空间宽裕
- 扣除工具碰撞球 (r=225mm) 后, 有效中心通道: X ∈ [−325, +325], Z ∈ [425, 925]

### 11.7 碰撞检测调用频率

| 场景 | 建议频率 | 说明 |
|------|---------|------|
| 路径采样 (RRT/PRM) | 每个采样点调用 1 次 | ~100-10000 次/秒 |
| 轨迹验证 | 每 1-2° 关节增量检查一次 | 确保连续段间无穿越 |
| 实时控制 | 250 Hz (4ms 周期) | 匹配伺服更新率 |

> `hrc_checkSelfCollision` 单次调用耗时约 10-50 μs (取决于障碍物数量), 足够满足实时需求。
